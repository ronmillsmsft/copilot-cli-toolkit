Get Started in 5 Minutes

Two ways to get started. Pick whichever feels right for you.
🧙
Path A: Guided
Setup Wizard
Recommended

Answer a few questions about yourself and your work. The wizard generates your config files, checks prerequisites, and walks you through every step of getting started. No command line experience needed.

Launch Setup Wizard

The wizard handles everything: prerequisites check, file generation, where to save them, how to open PowerShell, and your first command to type.

💻
Path B: Manual
Clone and Configure

Clone the repo, customize the instruction files, and initialize the memory system yourself. Best if you're comfortable with the command line and want full control over setup.

1 Clone the toolkit
git clone https://github.com/ronmillsmsft/copilot-cli-toolkit.git
2 Initialize the memory system
cd copilot-cli-toolkit/memory && python cli.py status
3 Launch Copilot CLI
gh copilot

Both paths get you to the same place: a personalized AI assistant running in your terminal. The wizard just walks you through it step by step.

Why This Matters

The Problem

Every AI conversation starts from zero. Your assistant forgets your preferences, your projects, your communication style. You repeat yourself constantly. You do the work while the AI just advises. Setup is manual, context is lost, and your assistant never actually learns who you are or how you work.

Key Signals

  • Persistent memory across all sessions
  • 115 preferences learned and applied automatically
  • Zero dependencies: Python + SQLite only
  • 5-minute setup to get started
  • Modular tiers: use only what you need
  • Real production use: not a demo, built for daily work

Why Copilot CLI?

GitHub Copilot shows up in many places. Here's why the CLI is the right tool for persistent, agentic workflows.

Copilot CLI is the only surface that gives you a persistent terminal agent with file system access, process execution, and the ability to build custom memory and automation layers. The other surfaces are great for their specific use cases, but CLI is where you build a true AI teammate.

Growth Template

  • ✓ Log conversation summary to memory
  • ✓ Save new preferences discovered
  • ✓ Save new insights (decisions, patterns, goals)
  • ✓ Update instruction files if workflows changed
  • ✓ Run /reindex to rebuild session index
Over time, this creates a compound effect. Each session builds on the last. After 30 sessions, your AI knows your tool preferences, stakeholder names, project history, and communication style without being told.

Design Principles

Action Over Discussion

Default to doing, not describing. Execute first, explain second.

Own End-to-End

Takes every task to completion. No half-finished handoffs.

Teach While Doing

Explains the why behind every decision and action taken.

Persistent Context

Every session builds on the last. Nothing is forgotten.
