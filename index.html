The Problem

Without Toolkit

  • AI forgets everything between sessions
  • Generic responses every time
  • You do the work, AI just advises
  • Manual setup every conversation
  • Scattered notes, lost context

With Toolkit

  • Remembers preferences, decisions, patterns
  • Personalized to your role and style
  • AI executes end-to-end, you review
  • Boot sequence loads context automatically
  • Persistent memory across all sessions

How It Works

Tier 1

Start Here

5 min

Give your AI a personality. Drop in a custom instruction file and your assistant greets you by name, matches your tone, and follows your rules.

What you get: Consistent AI behavior, personalized tone, custom rules Files: copilot-instructions.md
Tier 2

Build the Foundation

15 min

Add persistent memory. Your assistant remembers preferences, tracks decisions, and builds context session over session.

What you get: SQLite memory DB, preferences, insights, conversation history, FTS5 search Files: memory/cli.py, instruction files (SOUL.md, USER.md, AGENTS.md, etc.)
Tier 3

Automate Your Work

30 min

Turn routine tasks into one-line commands. Meeting prep, standup scans, action tracking, daily health checks.

What you get: 4 workflow tools (Action Tracker, Meeting Prep, Standup Prep, Daily Ops) Pattern: Each tool uses SQLite, runs from CLI, integrates with your AI assistant
Tier 4

Scale Up

1 hour Coming Soon

Multi-agent orchestration. Specialized agents with trust scoring, an ops dashboard, and MCP server integrations.

What you get: Agent engine, governance, Streamlit dashboard, MCP configs

Quick Start (5 Minutes)

1

Clone the repo

git clone https://github.com/ronmillsmsft/copilot-cli-toolkit.git

2

Run the Setup Wizard

Open starter/setup-wizard.html in your browser

3

Initialize memory

cd memory && python cli.py status

4

Launch Copilot CLI

Your assistant now has persistent memory

Prefer a guided experience? Open the Setup Wizard in your browser. No command line needed.

Growing Smarter Every Session

🚀

Boot

Memory loads, daily ops scan runs, AI greets you with context

⚡

Work

AI executes tasks, learns your preferences, tracks decisions

💾

Close

Session is logged, insights saved, memory updated

🔍

Index

Use /reindex to rebuild your local session index for fast retrieval

Session Close-Out Checklist

  • ✅ Log conversation summary to memory
  • ✅ Save new preferences discovered
  • ✅ Save new insights (decisions, patterns, goals)
  • ✅ Update instruction files if workflows changed
  • ✅ Run /reindex to rebuild session index
Over time, this creates a compound effect. Each session builds on the last. After 30 sessions, your AI knows your tool preferences, stakeholder names, project history, and communication style without being told.

Design Principles

🎯

Action Over Discussion

Default to doing, not describing what could be done

🔗

Own End-to-End

When the assistant takes a task, it owns it to completion

📖

Teach While Doing

Explain the why, not just the what

🧠

Persistent Context

Every session builds on the last

Requirements

  • GitHub Copilot CLI (with custom instructions support)
  • Python 3.10+
  • SQLite (included with Python)
  • No cloud services
  • No API keys
  • No external dependencies
